# SKILL.md 填空模板（指南第四部分）

按骨架选模板，替换 `[...]`，删除不需要的区块。所有模板共同要求：description
遵守四原则；正文 ≤500 行；关键步骤带检查点。

## 模板 A：线性流程型（Workflow + 线性/多阶段）

```markdown
---
name: [skill-name]
description: >
  [功能定义]。适用于[具体场景]，当用户[触发词1]、[触发词2]或[触发词3]时调用。
  包含[步骤A]、[步骤B]、[步骤C]。不适用于[排除场景1]和[排除场景2]。
---
# [Skill 名称]

## 前置条件
开始前检查（不满足则停止并告知原因）：
- [ ] [条件1]  验证：`[检查命令]`  预期输出：[预期]

## 主流程
### Step 1：[步骤名]
[祈使句指令。]
**为什么**：[目的/后果，帮 Agent 在未预料场景做对判断。]
> ⚠️ 易错点：[何时容易出错。]
**检查点**：运行 `[验证命令]`，预期[结果]；不符则停止修复后继续。

### Step 2：[步骤名]
[指令]  **为什么**：[原因]  **检查点**：`[验证命令]`（预期：[结果]）

### Step 3：[步骤名]
[指令]  **检查点**：`[验证命令]`

## 降级方案
如果 Step X 失败：1. [降级步骤] 2. [降级步骤] 3. `[验证命令]` 确认降级成功

## 常见问题
**Q：[常见问题]？** A：[可执行的解法。]
```

## 跨 session / 多阶段的叠加件

- 模式 4 接力棒：加 `## The Baton System`（接力棒文件规范）+ 6 步执行协议，
  其中 "Step 6: Prepare the Next Baton" 标 **CRITICAL + MUST**。
- 模式 5 多阶段：每个 Phase 统一 `Activities → Outputs → Decision Point
  （YES → 下一阶段 / NO → +X 天重做）`。
